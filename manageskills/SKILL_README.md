# ADK Course Workflow Automator

Author: Wissem Khlifi | Date: April 2026

Automated pipeline for Udacity Agentic AI Engineer
(Course 2-4) using ADK and Vertex AI.

> Each Udacity lesson has its own starter code,
> IntelliJ project, and GitHub repository.
> Copy only 2 scripts per new lesson project.
> Everything else is auto-generated by setup script.

For full procedures see OPS.md.

---

## Scripts in this folder

| Script | Purpose | When to run |
|--------|---------|-------------|
| setup_context_structure.py | Creates full folder structure and all context files | Once per new lesson project |
| ai_md_converter.py | Converts raw Udacity notes to compressed MD via Vertex AI | 3x per lesson (lesson, exercise, plan) |

---

## Slash Commands (invoke in Claude Code)

| Command | When | What it does | Output |
|---------|------|--------------|--------|
| /observe | After implementing | Adds BigQuery plugin, verifies streaming, runs LLM-as-Judge | Plugin in agent code, eval_results.md |
| /eval | After /observe | Local eval + BQ LLM-as-Judge + regression check | eval_results.md updated, pass/fail gates |
| /report | After /eval passes | Generates learning report from code + eval results | .claude/current/report.md |
| /security | Before commit | Scans credentials, packages, file access, MCP servers | Audit report PASS/WARN/FAIL |
| /review | Anytime | Reviews code against ADK patterns | Inline suggestions |
| /deploy | When ready | Deploys agent to Cloud Run | Deployment URL |
| /scaffold | New agent needed | Creates ADK agent scaffold | New agent files |

## Initial Setup (One-Time per Lesson Project)

### Step 1: Download Udacity starter code
Download for current lesson, open in IntelliJ.

### Step 2: Copy only 2 scripts

```bash
mkdir manageskills
cp /path/to/prev/manageskills/setup_context_structure.py manageskills/
cp /path/to/prev/manageskills/ai_md_converter.py manageskills/
cp /path/to/prev/README.md .
```

OPS.md and SKILL_README.md are auto-generated
by setup_context_structure.py -- no need to copy.

### Step 3: Install dependencies

```bash
uv init
uv add google-genai google-adk google-cloud-aiplatform
gcloud config set project agentic-2026-493108
gcloud auth application-default login
```

### Step 4: Run setup once

```bash
python3 manageskills/setup_context_structure.py --course 2 --lesson 9
# Replace 2 and 9 with your actual course and lesson numbers
```

This auto-generates:
- CLAUDE.md -- Claude Code CLI context
- GEMINI.md -- Gemini CLI context
- AGENTS.md -- cross-tool rules + security protocol
- .claude/current/ -- warm context templates
- .claude/commands/ -- 7 skills:
  /deploy, /eval, /scaffold, /review,
  /observe, /report, /security
- .agent/rules/ -- Antigravity handoff rule
- .agent/workflows/ -- /handoff slash command
- docs/archive/ -- cold storage
- docs/reference/ -- course-map.md, tool-stack.md
- manageskills/OPS.md -- all operational procedures
- manageskills/SKILL_README.md -- this file
- .gitignore -- excludes ephemeral + security files
- ~/.claude/settings.json -- idle timeout config

### Step 5: Initialize GitHub

```bash
git init
git remote add origin git@github.com:datai-wiss-dom/<repo>.git
git add .
git commit -m "L2 initial setup from Udacity starter code"
git push -u origin main
```

## Per-Lesson Workflow

### Step 6: Generate context files (before coding)

```bash
python3 manageskills/ai_md_converter.py --type lesson   --lesson L2
python3 manageskills/ai_md_converter.py --type exercise --lesson L2
python3 manageskills/ai_md_converter.py --type plan     --lesson L2
```

The lesson run automatically:
- Archives .claude/current/ to docs/archive/
- Deletes stale HANDOFF.md
- Updates CLAUDE.md, GEMINI.md, course-map.md

### Step 7: Implement exercise

```bash
claude
```

Prompt every time:

```
Implement the exercise following
.claude/current/lesson.md,
.claude/current/exercise.md and
.claude/current/plan.md.
Read the existing starter code first before
writing anything.
```

### Step 8: After implementing -- 4 slash commands

Run in order in Claude Code:

```
/observe   <- add BigQuery plugin + LLM-as-Judge
/eval      <- local eval + BQ quality gates
/report    <- generate learning report
/security  <- pre-commit security audit
```

Fix any FAIL from /security before committing.
Review report.md in IntelliJ before committing.

### Step 9: Commit and tag

```bash
git add .
git commit -m "L2 complete - report + eval results"
git tag L2-complete
git push origin main
git push origin L2-complete
```

### Step 10: Start next lesson
Download next lesson starter code from Udacity.
Open as new IntelliJ project. Repeat from Step 1.

## Antigravity Quota Interruption

```
/handoff  <- type in Antigravity chat
claude    <- reads HANDOFF.md, continues seamlessly
```

HANDOFF.md deleted automatically on next lesson run.
